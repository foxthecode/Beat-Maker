Update the CAPTURE feature in KickAndSnare.tsx.
Two changes vs the previous implementation:

1. Multi-bar detection: CAPTURE takes everything played (2-4 bars),
   not just 1 bar. Pattern length = first→last hit rounded up to
   nearest complete bar at detected BPM. No maximum bar limit.

2. CAPTURE button moves: it now sits on the same line as COUNTDOWN,
   to its right. Remove the full-width CAPTURE button that was placed
   above the looper banner.

The existing captureToSequencer already rounds up to nearest bar —
but it caps totalSteps at 32 when choosing the target time signature.
Replace that cap with a dynamic lookup that supports 1, 2, and 4 bars:

FIND (in captureToSequencer):

    totalSteps === 32

REPLACE WITH:
  // Pick the best matching time sig for the detected step count.
  // Prefer an exact match, fall back to 4/4 and let the sequencer
  // show the correct number of steps via the _steps override.
  const targetSig =
    TIME_SIGS.find(s => s.steps === totalSteps) ??
    TIME_SIGS.find(s => s.label === '4/4 ×2') ??
    TIME_SIGS.find(s => s.label === '4/4') ??
    TIME_SIGS[0];

Also remove any hardcoded cap on numBars — the existing
  const numBars = Math.max(1, Math.ceil(totalDurationMs / barMs));
is correct as-is and already supports any number of bars.
