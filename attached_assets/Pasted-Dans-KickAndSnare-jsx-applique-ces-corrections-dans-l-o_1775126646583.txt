Dans KickAndSnare.jsx, applique ces corrections dans l'ordre exact.
Ne touche pas au scheduler euclid, aux states de pattern, ni aux vues.

━━━ CORRECTION 1 — Grésillements audio Android ━━━

Dans la classe Eng, ajoute dans le constructor :
this._isMobile = /Android|iPhone|iPad/i.test(
  typeof navigator !== 'undefined' ? navigator.userAgent : '');

Dans engine.init(), remplace :
this.ctx = new (window.AudioContext||window.webkitAudioContext)();
Par :
this.ctx = new (window.AudioContext||window.webkitAudioContext)(
  this._isMobile ? { latencyHint:'playback', sampleRate:44100 } : {}
);

Après this.gOut.connect(this.ctx.destination), ajoute :
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'visible' &&
      this.ctx?.state === 'suspended') {
    this.ctx.resume();
  }
});

Dans engine.play(), après la ligne s.start(t), ajoute :
s.onended = () => {
  try { s.disconnect(); g.disconnect(); } catch(e) {}
};

━━━ CORRECTION 2 — Grésillements au changement de volume ━━━

Dans engine.uFx(id, f), remplace :
c.vol.gain.setTargetAtTime((f?.vol ?? 80)/100, t, 0.02);
Par :
const targetVol = (f?.vol ?? 80) / 100;
const transTime = this._isMobile ? 0.08 : 0.02;
c.vol.gain.setTargetAtTime(targetVol, t, transTime);

Et remplace :
if (c.pan?.pan)
  c.pan.pan.setTargetAtTime((f?.pan ?? 0)/100, t, 0.02);
Par :
if (c.pan?.pan)
  c.pan.pan.setTargetAtTime(
    (f?.pan ?? 0)/100, t, transTime);

━━━ CORRECTION 3 — Look-ahead adaptatif ━━━

Dans schLoop, remplace les deux occurrences de :
schRef.current = setTimeout(schLoop, 25);
Par :
schRef.current = setTimeout(schLoop,
  engine._isMobile ? 50 : 25);

Remplace la première occurrence de :
while(nxtRef.current < ct + 0.1)
Par :
while(nxtRef.current < ct + (engine._isMobile ? 0.2 : 0.1))

Remplace la première occurrence de :
while(ec.nextTime < ct + 0.1)
Par :
while(ec.nextTime < ct + (engine._isMobile ? 0.2 : 0.1))
